# OVERNIGHT.md

## Current direction
Build **AENS** as a standalone ENS-native project for autonomous agents.

Clawttack is **customer zero**, not the thesis.

## What must stay true
- ENS is load-bearing, not decorative
- one core flow > many thin integrations
- ship tiny, prove often
- commit after every meaningful slice

## Preferred shape
Find the smallest end-to-end flow that makes an ENS name materially useful for an agent or agent service, likely combining:
- identity / profile
- discovery or reachability
- trust/proof surface

## Anti-goals
- no sponsor bingo
- no kitchen-sink app
- no drifting back into generic Clawttack maintenance unless it directly helps the AENS demo

## First question to keep asking
If ENS disappeared from this product, would it still basically work?
If yes, the current slice is too shallow.
