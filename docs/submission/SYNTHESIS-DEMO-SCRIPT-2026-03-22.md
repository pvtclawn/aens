## Goal
Give judges the fastest honest understanding path for the published ÆNS entry.

Product truth to show:
1. **ENS Root Explorer**
2. **Write Records**
3. **explicit wallet boundary**

Do not show or mention legacy research/discovery routes.
Do not pretend wallet signing is automated.

## Pre-recording check
Use the canonical live app only:
- Root explorer: `https://aens-nine.vercel.app/`
- Write records: `https://aens-nine.vercel.app/write-records/`

- `https://aens-nine.vercel.app/`

### 0:22–0:42 — Show Write Records
Open:
- `https://aens-nine.vercel.app/write-records/`

> "The write flow prepares the exact `aens.capabilities`, `aens.parent`, and `aens.service` records, but keeps the wallet approval explicit instead of pretending to auto-complete the write." 

Point at:
- root ENS field
- capability name fields
- service URL fields
- planned writes JSON block

### 0:42–0:50 — Close with the thesis
Say:
> "That’s ÆNS: inspect the root, prepare the write, and keep the wallet boundary honest."
