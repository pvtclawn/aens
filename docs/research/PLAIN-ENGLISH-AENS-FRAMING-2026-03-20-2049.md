# Plain-English ÆNS framing (2026-03-20 20:49 UTC)

## Purpose
Freeze a plain-English description of ÆNS that a normal person can understand quickly.

Trigger:
- Egor explicitly said they still do not really get what ÆNS is

That means the project currently has a real product-framing problem, not just a technical-execution problem.

## Shortest honest description
### ÆNS lets an ENS name act like a verified profile that can publish official tools and pages.

## One-sentence version
### ÆNS helps someone use their ENS name as a verified home base for official sub-tools, pages, or agent capabilities.

## Slightly more concrete version
If `pvtclawn.eth` is the main identity, then names like:
- `research.pvtclawn.eth`
- `pay.pvtclawn.eth`
- `chat.pvtclawn.eth`

can act like official, parent-authorized capabilities of that identity.

The important part is not just “a subdomain exists.”
The important part is:
### the parent ENS identity officially authorizes it.

## Problem it solves in plain English
Without something like ÆNS, it is messy to tell:
- which page/tool really belongs to an agent or creator
- which link is the official research page
- which service is genuinely endorsed by the main identity

ÆNS tries to solve that by making the parent ENS identity explicitly endorse child capabilities.

## Homepage / hero candidate
### Verified ENS profiles with official tools.

## Submission tagline candidate
### Turn an ENS name into a verified home base for official apps, pages, and agent capabilities.

## "What it is" / "what it isn't"
### What it is
- a way to use ENS as a verified profile + capability publisher
- a way to mark which child services are official
- a way to make sub-tools/pages feel attributable, not random

### What it isn't
- not just a generic subdomain manager
- not a full walletless publishing system
- not proof that a service is fully live end-to-end by itself
- not a replacement for the actual app/tool behind the child name

## Best current mental model
### Domain name + verified profile + official sub-tools
That is the simplest human explanation right now.
